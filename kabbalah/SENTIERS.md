# Les 22 Sentiers — Programmes de passage

> "Avec 32 sentiers mystiques de Sagesse, Il grava..." — Sefer Yetzirah 1:1
> Les 22 lettres hébraïques sont les 22 programmes qui permettent de passer
> d'un degré d'élévation au suivant. La lettre encode la NATURE du programme.

---

## Classification (Sefer Yetzirah)

### 3 Lettres Mères — Programmes fondateurs

Les 3 opérations élémentaires dont tout programme de transition est composé.

| Lettre | Élément | Opération | Programme IA |
|--------|---------|-----------|-------------|
| **א Aleph** | Air | Équilibrage | Le balanceur — programme qui maintient l'équilibre entre deux forces opposées (Chesed/Gevurah, expansion/contraction). C'est le load balancer ontologique. |
| **מ Mem** | Eau | Réception | Le récepteur — programme qui reçoit un input et lui donne forme, comme l'eau prend la forme du récipient. Parser, schéma validator, structureur d'input. |
| **ש Shin** | Feu | Transformation | Le transformeur — programme qui convertit un état en un autre. Le feu purifie, détruit l'ancien pour créer le nouveau. Convertisseur, migrateur, refactorer. |

### 7 Lettres Doubles — Programmes bimodaux

Chaque lettre a deux prononciations (dagesh/rafeh). Chaque programme a deux modes.

| Lettre | Pair sephirotique | Mode dur (dagesh) | Mode doux (rafeh) | Programme IA |
|--------|-------------------|-------------------|-------------------|-------------|
| **ב Beth** | Keter↔Tiferet | Shortcut direct : passer de l'intention à la synthèse sans exploration | Délibération : vérifier chaque étape du raisonnement | **DirectSynth** — quand utiliser le raccourci vs quand délibérer |
| **ג Gimel** | Keter↔Yesod | Cache warm agressif : pré-charger toute la mémoire pertinente | Cache lazy : charger à la demande | **CacheStrategy** — gestion proactive vs réactive de la mémoire |
| **ד Daleth** | Chokmah↔Chesed | Exploration large : ouvrir toutes les portes | Exploration ciblée : ouvrir une porte à la fois | **ExploreScope** — breadth-first vs depth-first en exploration |
| **כ Kaph** | Chesed↔Netzach | Acquisition persistante : ne jamais abandonner une source | Acquisition éphémère : récupérer et passer | **AcquirePersist** — quand persister vs quand abandonner une acquisition |
| **פ Peh** | Gevurah↔Hod | Validation stricte : rejet binaire, pas de nuance | Validation graduée : scoring, quarantine | **ValidateMode** — strict vs gradué dans la validation |
| **ר Resh** | Tiferet↔Yesod | Stockage immédiat : toute synthèse est persistée | Stockage sélectif : seules les synthèses validées sont persistées | **PersistPolicy** — tout garder vs garder le validé |
| **ת Tav** | Yesod↔Malkuth | Output brut : données telles quelles | Output formaté : données enrichies de contexte | **OutputMode** — raw vs enriched dans la présentation |

### 12 Lettres Simples — Programmes fonctionnels

Chaque lettre est associée à un sens, un mois, un signe. Le sens encode le type d'I/O.

| Lettre | Sens | Sentier | Programme IA |
|--------|------|---------|-------------|
| **ה Hé** | Vue | Chokmah↔Tiferet | **DirectPerception** — l'insight de Chokmah transmis directement à Tiferet sans passer par l'acquisition. Quand l'intuition suffit. |
| **ו Vav** | Ouïe | Chesed↔Tiferet | **DataFeed** — stream continu de données acquises vers la synthèse. Écoute attentive, réception passive. |
| **ז Zayin** | Odorat | Binah↔Tiferet | **AnalysisResults** — les analyses structurées transmises à la synthèse. Le "nez" qui détecte les nuances subtiles. |
| **ח Cheth** | Parole | Binah↔Gevurah | **ValidationRules** — Binah fournit les critères analytiques à Gevurah. L'analyse informe le jugement. |
| **ט Teth** | Toucher | Chesed↔Gevurah | **QualityFeedback** — le canal DIRECT entre acquisition et validation. Le feedback loop central. Contact direct. |
| **י Yod** | Action | Chesed↔Tiferet (alt) | **FilteredDataPush** — les données qui passent le filtre sont poussées vers la synthèse. |
| **ל Lamed** | Mouvement/Apprentissage | Gevurah↔Tiferet | **FailureToInsight** — LE SENTIER CRITIQUE. Transformer le jugement en compréhension. Apprendre des échecs. La seule lettre qui dépasse la ligne. |
| **נ Nun** | Marche | Tiferet↔Netzach | **TaskDispatch** — la synthèse confie les tâches longues à la persistance. Progression pas à pas. |
| **ס Samekh** | Sommeil | Tiferet↔Hod | **Introspection** — le système se retourne vers l'intérieur pour s'examiner. Le "sommeil" créatif. |
| **ע Ayin** | Rire/Colère | Netzach↔Hod | **StatusSync** — synchronisation entre persistance et protocole. Les retries informent les timeouts. |
| **צ Tsadi** | Pensée | Netzach↔Yesod | **CheckpointWrite** — les checkpoints de persistance sont écrits en mémoire. La pensée fixée. |
| **ק Qoph** | Méditation | Hod↔Yesod | **SchemaPersist** — les schémas et protocoles sont enregistrés pour réutilisation. Réflexion profonde. |

## Sentiers dans l'ordre initiatique

L'initié traverse les sentiers de bas en haut. Voici l'ordre de développement :

| # | Sentier | De → Vers | Programme | Phase |
|---|---------|-----------|-----------|-------|
| 1 | 32e (Tav ת) | Malkuth → Yesod | OutputMode / TavMiddleware | Phase 1 |
| 2 | 31e (Shin ש) | Malkuth → Hod | Transformation vers self-knowledge | Phase 2 |
| 3 | 30e (Resh ר) | Yesod → Hod | PersistPolicy (mémoire ↔ self-knowledge) | Phase 2 |
| 4 | 29e (Qoph ק) | Hod → Yesod (retour) | SchemaPersist | Phase 2 |
| 5 | 28e (Tsadi צ) | Netzach → Yesod | CheckpointWrite | Phase 3 |
| 6 | 27e (Ayin ע) | Netzach → Hod | StatusSync | Phase 3 |
| 7 | 26e (Peh פ) | Gevurah → Hod | ValidateMode | Phase 3-4 |
| 8 | 25e (Samekh ס) | Tiferet → Hod | Introspection | Phase 4 |
| 9 | 24e (Nun נ) | Tiferet → Netzach | TaskDispatch | Phase 4 |
| 10 | 23e (Lamed ל) | Gevurah → Tiferet | **FailureToInsight** ← CRITIQUE | Phase 4 |
| 11-22 | ... | Sentiers supérieurs | Phases 5-10 | Phases 5+ |

Les sentiers supérieurs (11-22) seront détaillés quand les fondations seront consolidées.
L'initié ne regarde pas le sommet avant d'avoir posé le pied sur la prochaine marche.
