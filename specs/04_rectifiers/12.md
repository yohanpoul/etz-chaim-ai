# Rectifier 12 — Probe Noise

## Failure pattern

A probe (other rectifier) fires more than N times in 24h with > 70% of
firings classified post-hoc as false positives. Indicates probe
threshold needs tightening.

```python
{
    "rectifier_id": "r12",
    "pattern": "probe_noise",
    "metrics": {
        "noisy_probe_id": str,
        "firings_24h": int,
        "false_positive_rate": float,
    },
}
```

## Actions

- `observe` : log only.
- `suggest` : emit `r12_action_proposed` recommending threshold
  tightening for the noisy probe.
- `act` : tighten the probe's primary threshold by 10%; cap cumulative
  tightening at 50% of original (prevents probe death).

## Invariants

1. Cumulative threshold tightening capped at 50% of original.
2. Tightening reversible via `restore_threshold(probe_id)`.
3. Never disables a probe entirely.
4. Self-exemption : r12 cannot rectify itself (no recursive tightening).

## Tests

- Noisy probe with FPR > 0.7 and > 20 firings → deviation.
- Quiet probe → no deviation.
- `act` tightens within 10% step, respects cumulative cap.
- Self-firing of r12 produces a flagged warning, not a self-tightening.
