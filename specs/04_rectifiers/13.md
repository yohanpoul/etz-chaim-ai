# Rectifier 13 — Trace Amnesia / Stale Causal Claims

## Failure pattern

The persistent trace coefficient on a faculty has been zero for >= N
cycles, OR causal claims marked `confounders_controlled = false` are
older than 30 days without follow-up. Existing implementation as
`StaleClaimsRectifier` in `probes/` covers the second sub-pattern.

```python
{
    "rectifier_id": "r13",
    "pattern": "trace_amnesia_or_stale_causal",
    "metrics": {
        "zero_trace_cycles": int,
        "stale_causal_claims": int,
        "oldest_claim_age_days": float,
    },
}
```

## Actions

Existing `act` mode marks stale causal claims as `abandoned`. Future
extension : also detect zero-trace faculties and trigger a warm-up
boost via the cross-coupling layer.

## Invariants

1. Marking `abandoned` is reversible via `unabandon(claim_id)`.
2. Warm-up boost respects the plateau ceiling (rectifier 9 invariant).
3. Idempotent.

## Tests

Existing tests in `probes/tests/`. New work : extension to detect
zero-trace pattern + neutral re-export through `etzchaim/probes/`.
