# Rectifier 01 — Contradiction Resolution

## Purpose

Rectifier triggered when two memory entries are tagged contradictory and
neither has been investigated for over 24 hours. Proposes (in `suggest`
mode) or applies (in `act` mode) a synthesis pass : either elevate the
better-supported entry, mark both as parked-pending-evidence, or open
a `dissensus` ticket for active probing.

Distinguishes from `failure_insight_conversion` (rectifier 5) by acting
on memory contradictions, not on policy adjustments after live failures.

## Public API

```python
class ContradictionResolutionRectifier(BaseRectifier):
    def detect(self, tree: dict) -> list[Deviation]: ...
    def rectify(self, deviations: list[Deviation], mode: RectificationMode) -> list[Event]: ...
```

## Action proposed

Three modes :
- `observe` : log the deviation only.
- `suggest` : emit `r01_action_proposed` event with the chosen action.
- `act` : execute the chosen action automatically (only after 4 weeks of
  observation showing no false positives).

Three actions selectable :
- `elevate_winner` : when one entry has `confidence > 0.8` and the other
  `< 0.4`, elevate the winner and mark the loser archived.
- `park_pending_evidence` : when both have similar confidence, mark both
  as `parked` and open a follow-up question in the next probing cycle.
- `open_dissensus` : when contradiction is conceptually deep, open a
  full dissensus ticket via the Tension faculty.

## Invariants

1. **Never deletes** : rectifier never removes memory entries; it only
   changes their status flags.
2. **Idempotent** : applying the same rectification twice yields the same
   state.
3. **Reversible in `act` mode** : every action records an undo recipe in
   the daemon log so a human reviewer can roll back.

## Tests

`tests/test_rectifier_01.py` must cover :

- Two contradicting memories aged 25h → deviation detected.
- Two contradicting memories aged 1h → no deviation.
- `act` mode applies the right action for each of the three branches.
- Action is idempotent (apply twice → same state).
- Undo recipe correctly reverses each action type.
