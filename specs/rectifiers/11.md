# Rectifier 11 — Calibration Drift

## Failure pattern

The 49 calibration parameters (7×7 matrix) drift more than threshold
percent from their post-onboarding baseline, in aggregate. Indicates
parameter creep that may degrade overall system behavior.

```python
{
    "rectifier_id": "r11",
    "pattern": "calibration_drift",
    "metrics": {
        "params_above_threshold": int,
        "max_individual_drift_pct": float,
        "rms_drift": float,
    },
}
```

## Actions

- `observe` : log only.
- `suggest` : emit `r11_action_proposed` listing drifting params and
  recommending partial restoration.
- `act` : restore drifting params toward baseline by 50% of the drift
  amount; log a calibration audit entry.

## Invariants

1. Restoration is partial (50% pull) — never snaps back to baseline.
2. Audit log entry includes pre/post values for every modified param.
3. Reversible via the audit log replay.

## Tests

- RMS drift > threshold → deviation detected.
- All params within band → no deviation.
- `act` modifies only the drifted params, by exactly 50% pull.
- Audit log lets a human reviewer reverse the action.
