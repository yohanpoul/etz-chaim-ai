# Rectifier 10 — Recursive Analogy

## Failure pattern

Analogy chain where each analogy's source is itself an analogy, with depth
> N. Indicates the Exploration faculty has recursed without grounding,
producing a chain disconnected from primary observations.

```python
{
    "rectifier_id": "r10",
    "pattern": "recursive_analogy",
    "metrics": {
        "max_chain_depth": int,
        "deepest_chain_id": str,
        "non_grounded_chains": int,
    },
}
```

## Actions

- `observe` : log only.
- `suggest` : emit `r10_action_proposed` recommending grounding pass on
  deepest chains.
- `act` : truncate chains at depth > 3; keep first 3 hops, archive the
  rest with `truncated_recursive` flag.

## Invariants

1. Truncation never deletes; archived hops queryable for replay.
2. Depth limit configurable but capped at 5 in any case.
3. Idempotent : second pass over already-truncated chains is no-op.

## Tests

- Chain depth 5 → deviation, truncate to 3.
- Chain depth 2 → no deviation.
- Truncation reversible via `restore_chain(chain_id)`.
- Cap at 5 enforced regardless of config.
