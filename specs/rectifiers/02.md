# Rectifier 02 — Memory Bloat

## Failure pattern

Memory store grows past N entries with median confidence < 0.3 and access
count < 2 over 30-day window. Indicates accumulation of low-value entries
crowding out signal.

```python
{
    "rectifier_id": "r02",
    "pattern": "memory_bloat",
    "metrics": {
        "total_entries": int,
        "low_confidence_low_access": int,
        "ratio": float,
    },
}
```

## Actions

- `observe` : log only.
- `suggest` : emit `r02_action_proposed` with prune candidates.
- `act` : archive entries with confidence < 0.3 AND access_count < 2 AND
  age_days > 30. Keep undo recipe (re-promote on demand).

## Invariants

1. Never deletes; only archives. Archive table preserves entries.
2. Idempotent : applying twice yields same archived set.
3. Reversible : undo recipe restores entries to active state.
4. Bounded : never archives more than 25% of active entries in one pass.

## Tests

- Bloat above threshold → deviation detected.
- Bloat below threshold → no deviation.
- `act` archives only entries matching all three conditions.
- 25% cap enforced.
- Undo restores the archived set exactly.
